  正典は `SPECIFICATION.html` のみ。本書と正典が食い違う場合は正典が優先する。

## 0. ビジョン

> Ajisai を表計算ベースのアプリに作り替える。
> 現在の Playground は、Google Spreadsheet における Apps Script エディタの位置づけに移す。

表計算ソフトの強みは次の 2 点に集約される:

1. **セルがテキストエディタとして機能する** — 入力の敷居が極端に低い。
2. **セルがそのまま GUI になる** — 書いた式の結果が常に見えていて、値の変更が即座に伝播する。

「プログラミング学習の開始から実用物の作成までの距離が極端に短い」というのは
この 2 点の帰結であり、Ajisai の設計思想（構造が見える・部分失敗が見える・値が正確）と
本質的に相性がよい。以後、新しい主画面を **Sheet ビュー**、
現 Playground 相当を **Editor ビュー**（Apps Script 相当）と呼ぶ。

### Ajisai が表計算の土台として持つ固有の強み

| 表計算の概念 | Excel/GSheet | Ajisai での対応物 | 優位性 |
| --- | --- | --- | --- |
| セルの値 | 浮動小数点 | 正確な有理数／連分数 | `0.1+0.2` 問題が存在しない。金額計算で丸め誤差ゼロ |
| エラー値 | `#DIV/0!` など不透明な定数 | 理由つき `NIL`（Bubble） | 絶対値でなく診断メタデータ付きの値として下流に流れる |
| `IFERROR` | 関数でラップ | `^`（VENT）1 語 | パイプライン末尾で一度だけ受ければよい |
| 判定不能 | 存在しない（誤答になる） | `UNKNOWN`（Stagnation） | 予算内で決まらない比較が「未決」として見える |
| 範囲 (A1:A3) | 専用のRange型 | Vector そのもの | 範囲＝第一級の値。MAP/FILTER/FOLD が直接効く |
| 動的配列のスピル | 後付け機能 | Vector 値の自然な表示 | 言語側は何も足さなくてよい |
| ユーザー定義関数 | Apps Script / VBA | ユーザーワード（DEF） | 言語が一枚岩。シートとスクリプトで方言が分かれない |

最後の行が本プランの核心である。Excel は「セルの式言語」と「拡張言語（VBA/GAS）」が
別言語だが、Ajisai では **セルの式もワード定義も同じ Ajisai** にできる。

## 1. 核心の設計判断: シート＝辞書、セル＝ワード

新しい概念をほとんど導入しない。既存の語彙システムに表計算を写像する:

| 表計算の概念 | Ajisai の既存概念 |
| --- | --- |
| シート | ユーザー辞書（例: `SHEET` 辞書。複数シート＝複数辞書） |
| セル A1 | `SHEET` 辞書内のワード `A1` |
| セルの数式 | ワード本体（`{ ... }`） |
| セル参照 | ワード呼び出し（修飾名 `SHEET@A1`） |
| 再計算 | 逆依存インデックスによる従属ワードの無効化・再評価 |
| 循環参照エラー | 依存グラフ上のサイクル検出（ホスト側） |

この写像が成立する根拠は、ランタイムが **既に** 持っている機構にある:

- ワードの依存関係は forward/reverse 両方向で維持されている
  （`Interpreter::dependents` 逆引きインデックス、
  `docs/dev/dependents-inverted-index-reads.md`）。
- `collect_transitive_dependents` が「再定義の影響範囲（BFS 閉包）」を
  O(影響集合) で返す。これは表計算の dirty-set 計算そのものであり、
  同ドキュメントに「後のキャッシュ無効化ステージが使う」と明記されている。
  本プランがまさにそのステージである。
- 修飾名 `X@WORD` は解決はしごを迂回して辞書に直行する
  （`resolve_word.rs`）。セル参照を `SHEET@A1` に正規化すれば、
  Core 語やユーザーワードとの名前衝突問題が構造的に消える。
- ワードの内容アドレス識別（§8.6, `collect_word_identities`）により、
  同一数式のセルは同一の内容 ID を持つ。将来の共有式最適化の足場になる。

**言語コアへの変更は最小**で済む。表計算はホスト（GUI/TS 層）の機能であり、
これはリポジトリの既存原則 — 効果はホストコマンドとして境界（Gate）を渡る、
純粋コアは host 機能を持たない — と整合する。

## 2. セル評価モデル

### 2.1 評価規約

- セルの数式は **空のスタック** で評価を開始し、**共有辞書**（Core + import 済み
  Module + ユーザー辞書 + SHEET 辞書）を参照する。
- 評価後の **最終スタックがセルの値**。慣例として 1 値（多くは `[ ... ]` の
  Vector か scalar）。複数値が残った場合は第一段では「スタック表示のまま
  セル内に縦に描画」し、リントで注意表示する（エラーにはしない — 遊べる余地を残す）。
- 表示は既存の `value-formatter` / `value-latex`（KaTeX、opt-in）を再利用。
  `3/1` の正準表示・分数の KaTeX 表示・NIL/UNKNOWN の表示資産がそのまま効く。

### 2.2 失敗と未決の表示（Ajisai らしさが最も出る箇所）

- `NIL` セル: Bubble を理由つきで表示（例: 小さく `NIL · divisionByZero`）。
  ホバー/選択で `semantics.absence` の診断を Editor ビュー同様に展開。
- `UNKNOWN` セル: 判定未決として表示（`agreedPrefix` 等の診断つき）。
- チャネルエラー（不正形）: セルはエラー状態になり、`diagnosis.why/where` を
  セル詳細に表示。**エラーは値にならない** という言語の原則をそのまま維持する。

### 2.3 純粋性ポリシー（Gate の適用）

セル評価は **効果なし（outward gate 閉）** を第一段の規約とする:

- `PRINT` / `IO@*` / `SERIAL@*` / `MUSIC@PLAY` 等のホスト効果語はセル内では
  チャネルエラー（`fixProgram` 診断）。効果を使いたいコードは Editor ビューで実行する。
  既存の `is_safe_preview_word` の判定系を評価前チェックに転用できる。
- 実行ステップ予算・比較予算は **セル単位** に適用（既存の water level 機構）。
  暴走した 1 セルがシート全体を巻き込まない。

### 2.4 再計算エンジン

1. セル編集 → ホストが `SHEET` 辞書へ再定義（依存者が居るセルの上書きは通常運転
   なので、DEF の再定義ガードを通らない **ホスト専用の定義 API**、または強制修飾
   `!` 相当の経路を使う）。
2. `collect_transitive_dependents('A1')` で dirty set を取得
   （現在 Rust 内部のみ。**WASM export の追加が必要** — 数行の追い出しで済む）。
3. ホスト側でサイクル検出（dirty set 内のトポロジカルソート）。サイクルは
   評価せず「循環参照」としてセルをエラー表示。ワードの再帰は言語として合法だが、
   **セル間参照の循環は表計算の慣例どおりホストが拒否**する（深さ上限で止まる
   診断エラーに任せるより UX が良い）。
4. トポロジカル順に再評価。同一レベルのセルは独立なので、既存の
   worker manager + interpreter snapshot（`src/workers/`）で **並列評価** できる。
   これは `implicit-parallelism-roadmap` の方向性とも一致する（第一段は直列で可）。
5. 各セルの評価結果はホスト側キャッシュ（TS の `Map<CellAddr, Value>`）に保持。
   ワード本体は再評価のたびに走るのが言語側の意味論だが、セル値のキャッシュは
   ホストの表示状態なので意味論を汚さない。

## 3. セル参照の構文

### 3.1 数式の表面（Sheet ビュー内のみの規約）

- グリッド内でセルに `= ...` で始まるテキストを入れると数式、それ以外は
  リテラル値（数値として parse できれば数値、できなければ文字列）。
  これは表計算の万国共通規約に従う。`=` はホスト側の目印であり言語には入れない。
- 数式内の **裸の `A1` 形トークン**（`[A-Z]+[0-9]+`）はホストの前処理で
  `SHEET@A1` に書き換えてからインタープリタへ渡す。ユーザーが `SHEET@A1` と
  書いても同じ。多シート時は `SHEET2@B3` のような辞書修飾で越境参照。
- 範囲 `A1:A3` はホスト前処理で `[ SHEET@A1 SHEET@A2 SHEET@A3 ]` に展開
  （矩形範囲は入れ子 Vector）。範囲＝Vector なので、その先は
  `SUM` 系も `{ ... } MAP` も言語の既存語彙がそのまま効く。

例（すべて既存の言語仕様の範囲内で動く形）:

```ajisai
= A1 B1 +                      # 2 セルの和
= A1:A10 [ 0 ] { + } FOLD      # 合計（後述の SUM ワードで短縮可）
= A1:A10 { [ 100 ] > } FILTER  # 100 超の値だけの Vector（スピル候補）
= B2 [ 0 ] / ^ [ 999 ]         # 0 除算の Bubble を ^ で 999 に置換
```

### 3.2 衝突ポリシー

- `A1` 形はセル参照が **数式表面でのみ** 優先される。前処理での書き換えなので
  言語の解決はしごには一切手を入れない。Editor ビューのコードでは従来どおり
  `A1` はただのワード名（ただし紛らわしいのでリントで注意）。
- 逆方向の保護: Sheet ビューから `A1` 形の名前を **セル以外の用途で DEF する**
  ことはホストが拒否する。

### 3.3 スピル（後続フェーズ）

Vector 値のセルは、右/下の空セルに **表示だけ** あふれる（値の所有は起点セル）。
Excel の動的配列と同じ見え方だが、言語側には何も追加しない。
第一段では「セル内に Vector をそのまま描画」で出荷し、スピルは Phase 5。

## 4. Editor ビュー（旧 Playground）の位置づけ

Google Spreadsheet の「拡張機能 > Apps Script」に対応する動線にする:

- Sheet ビューがアプリのホーム。ヘッダのボタン（例: `Words` / `Editor`）で
  Editor ビューへ切り替え（モーダルでなくビュー切替。モバイルは既存の
  panel selector の 1 モードとして追加）。
- Editor ビューは現在の 4 パネル（Input / Output / Stack / Dictionary）を
  **ほぼそのまま維持**。ここで `{ ... } 'TAX' DEF` したワードは即座に
  シートの数式から `= A1 TAX` のように呼べる。
- ワードを再定義したら、そのワードに依存するセルも dirty になる。
  これも同じ逆依存インデックスで賄える（ワード → 従属セルの閉包）。
- Dictionary パネルの「ワードをクリックして挿入」は、Sheet ビューの
  数式編集中にも効かせる（セル編集こそ語彙探索が要る場面）。

こうして「セルで書き始め、複雑になったら Editor でワードに切り出し、
セルは `= A1:A10 TAX-TOTAL` のように短くなる」という、
Apps Script のカスタム関数と同じ成長曲線が一本の言語の中で成立する。

## 5. アーキテクチャとモジュール構成

```text
Rust core（変更は最小）
  └ WASM 追加 export:
      - collect_transitive_dependents(name)   … dirty set 取得
      - define_word_forced(name, body) 相当    … セル再定義用（ガード迂回のホストAPI）
      - (任意) execute_pure(code)             … 効果語を評価前拒否する純評価入口

TypeScript（主戦場）
  src/sheet/                     … 新設。UI 非依存の純ロジック + vitest
    ├ cell-address.ts            … A1 形式 ⇔ (row, col)、範囲展開
    ├ formula-preprocessor.ts    … `=` 判定、A1→SHEET@A1 書換、範囲展開
    ├ sheet-engine.ts            … セル表・dirty 伝播・トポソート・循環検出
    ├ sheet-evaluator.ts         … worker 経由のセル評価・キャッシュ
    └ sheet-persistence.ts       … 文書モデルの保存/復元
  src/gui/grid/                  … 新設。グリッド UI（仮想スクロール、自前実装）
    ├ grid-view.ts               … 描画・選択・キーボードナビゲーション
    ├ cell-editor.ts             … セル内エディタ（code-input-editor を流用）
    └ cell-renderer.ts           … value-formatter / value-latex / NIL・UNKNOWN 表示
  既存 src/gui/                  … Editor ビューとして温存。layout-controller に
                                   Sheet/Editor のビュー切替を追加
```

- グリッドは外部依存を増やさず自前実装（このリポジトリの依存方針に合わせる）。
  第一段は固定 1000 行 × 26 列程度＋仮想スクロールで十分。
- 評価は既存の execution worker 経路に載せ、メインスレッドを塞がない。
  abort（Escape）もそのまま効く。

### 5.1 永続化

- **セルの数式と値の意味はワードとして** 既存のユーザーワード永続化
  （IndexedDB / Tauri JSON、`interpreter-state-persistence.ts`）に自然に載る。
- **グリッド固有のメタデータ**（列幅・書式・選択状態・シート一覧）は
  別ストア（同じ platform adapter 経由の `sheet-document` キー）に保存。
- 入出力: 第一段は自己完結の JSON（数式・値・メタデータ）。CSV import/export を
  Phase 4 で追加（export は正準表示文字列、import はリテラル値として取り込み）。
  遅延連分数の表示文字列は再パース禁止（既存規約）なので、値の交換は
  JSON プロトコル形式を使う。

## 6. 段階的マイルストーン

各フェーズは独立にレビュー・マージ可能な粒度にする。

| --- | --- | --- |

| **5** | 磨き込み: スピル表示、範囲選択→式への参照挿入、worker 並列再計算、KaTeX セル表示、モバイルレイアウト | 任意（各項目独立に出荷可能） |

Phase 3 で URL パラメータ（例: `?view=editor`）により Reference の
「Playground で開く」リンクを Editor ビューへ直行させる。

## 7. テスト計画

- `src/sheet/` は DOM 非依存の純ロジックとして設計し、vitest で
  アドレス変換・前処理書換・トポソート・循環検出・dirty 閉包を単体テスト。
- セル評価の意味論は既存の言語 conformance を **一切変えない**
  （セル評価＝空スタックでの通常評価 + 前処理書換、であることをテストで固定）。
- `gui-interpreter-test-cases.ts` に Sheet ビューの結合ケースを追加
  （編集→再計算→NIL 表示→`^` 復旧、の一連）。
- Rust 側の追加 export は既存の dependents cross-check
  （`debug_assert_eq!` によるスキャン照合）の傘の下に入る。

## 8. 未決事項（Phase 0 で決める）

1. **複数値スタックのセル**: 縦積み表示（推奨・遊べる）か、単一値を強制か。
2. **辞書名とシート名**: 既定シートの辞書名（`SHEET` / `S1` / ユーザー命名）。
   シートのリネームは辞書リネームになるため、参照の一括書換が要る。
3. **効果語の扱い**: 全面拒否（推奨）か、`MUSIC@PLAY` 等の再生系だけ
   「セル選択時に手動発火」ボタンとして許すか（セル＝GUI の思想的には後者が面白い）。
4. **裸 `A1` 書換の適用範囲**: 数式表面のみ（推奨）で確定してよいか。
5. **アプリ名/表記**: Sheet ビューの呼称（例: そのまま「Ajisai」、エディタ側を
   「Words」と呼ぶ等）。ヘッダ・README・Reference の表記に波及。

## 10. この設計が守る Ajisai の原則

- **正典不可侵**: 言語意味論に変更なし。表計算はホスト面（Gate の外側）の機能。
  正典に足すとしても非規範の Host Surface 記述のみ。
- **値の完全性第一**: セルに浮動小数点を持ち込まない。丸めは表示のみ。
- **部分失敗の可視性**: `#VALUE!` 的な不透明定数を作らず、理由つき NIL /
  UNKNOWN / チャネルエラーの三区分をセル表示にそのまま昇格させる。
- **AI-first**: セルの数式・依存グラフ・診断はすべて機械可読
  （ワード本体 + `collect_dictionary_dependencies` + JSON 診断）。
  AI エージェントが「シートを読む・直す」ことが最初から可能な表計算になる。
