# Ajisai 比較・数値経路 高速化ロードマップ（兆し①②③）

> 非正準・設計検討ノート。Ajisai の意味論は `SPECIFICATION.html` のみが定義する。
> 本ノートは観察（`ajisai-structure-mathematical-observations.md`）から派生した
> 高速化候補3件を、**測定データ**と**検証方針**つきで整理し、効率的な実装順を示す。
> いずれの最適化も「決定結果は厳密路と完全一致」を不変条件として固定し、
> 短絡率・早決定率を回帰指標として測る、という同じ型で守る。

## 方針：高速化と回帰防止は両輪

各最適化は次の二点を必ずセットにする。

1. **不変条件（ハード assert）**：最適化が決定した順序は、厳密路（RCF 参照／既存 NICF）が
   決定する順序と*必ず*一致する。最適化は「決定を*速める*」だけで「決定を*変えない*」。
2. **効果指標（回帰メトリクス）**：短絡率・早決定率を決定論的コーパスで測り、劣化したら
   テストが落ちるよう下限／上限を固定する。`nicf_order_invariant_and_early_decision_rate`
   と `interval_filter_reach_*` が既にこの型の先例。

## ① 区間プレフィルタ — 測定先行の結論

**狙い**：厳密実数比較の定石「包含区間が交わらなければ O(1) で順序確定、交わるときだけ厳密展開」。
Ajisai 固有の利点は、Gosper/NICF エミッタが項を出す際に既に値域端点
（Möbius は `v(∞)=a/c`, `v(1)=(a+b)/(c+d)`；双一次は4隅）を評価しており
（`continued_fraction.rs` の NICF emit 条件）、**包含区間が係数からただで取れる**こと。
別途エンクロージャ場を持たずに済む。

**測定（`interval_filter_reach_*`、決定論的シード）**：

| コーパス | r=1 で分離 | ≤3 項で分離 | NICF との比較 |
| --- | --- | --- | --- |
| 独立ランダム（典型データ、4000 ペア） | **89.8%** | 98.3% | — |
| 近接ペア（敵対的、4000 ペア） | 0% | ~2% | フィルタが NICF より遅い 88%、早い 0% |

**結論（期待値の正確化）**：

- 典型データでは **約 90% の比較が単一の区間判定（r=1）で即決**でき、ストリーム比較機構に
  入らずに済む。これが ① の価値の中心（共通ケースの O(1) 化）。
- **近接・等値ペアでは区間フィルタは NICF より弱い**（収束区間の完全分離は NICF の項分岐より
  多くの項を要する）。よって ① は **NICF の置き換えではなく前段フィルタ**であり、分離を
  証明できない場合は必ず既存 NICF へ落とす。
- 区間フィルタは「分離を*証明*したときだけ」短絡するので、厳密性は一切損なわない（測定テストは
  「フィルタが誤った向きに分離しない」を assert で固定済み）。

**実装方針**：`cmp_with_budget_tracked` の冒頭（代数的短絡の直後）に、両オペランドの
値域端点から包含区間を構成し交差判定する前段を置く。交差（=分離不能）なら現行の NICF ループへ。
予算契約（`COMPARE-WITHIN` の agreedPrefix）には影響しない——フィルタは決定済みになる比較しか
短絡せず、U を生むのは従来どおり NICF 経路のみ。

## ② Lagrange 全周期キャッシュ — 測定により見送り

**当初の狙い**：`sqrt_cf_period`（√D の `(前周期, 周期)`）をキャッシュし、N 番目の部分商を
周期への添字で O(1) 化、同一 surd の反復展開コストを償却する。

**実現可能性測定（`sqrt_period_length_growth_measurement`）**：周期長は radicand とともに増大する。

| D の桁 | 平均周期長 | 最大周期長 |
| --- | --- | --- |
| ~1e1 | 2.4 | 5 |
| ~1e3 | 19.8 | 48 |
| ~1e5 | 161 | 370 |
| ~1e6 | 202 | 866 |

**見送りの理由（コスト profile のミスマッチ）**：

- 全周期キャッシュの事前計算・メモリは **O(周期長)**。周期長は概ね √D log D で増大するため、
  キャッシュが最も効いてほしい**大 D（各 (P,Q) ステップが重い BigInt 演算）でこそ周期が長大**になり、
  キャッシュ生成・保持が高コスト化する。
- 加えて `sqrt_cf_period` の重複状態探索は `seen.iter().position(...)` による **O(周期²)**。
  D≈1e9 では周期計算自体が事実上ハングし（測定中に 2 分以上 CPU 100% を実測）、
  **キャッシュを作る段階で破綻**する。
- 逆に周期が短い**小 D は既に i128 `SqrtSmall` 高速路**で展開が安く、キャッシュの上積みは小さい。
- そもそも比較・算術は項を 0 から順次消費するため O(1) ランダムアクセスの恩恵が薄く、
  per-term も既に O(1)。

→ **全周期キャッシュは実装しない。** もし将来 ② を追うなら、周期長に依存しない
**有界プレフィックス memo**（radicand ごとに先頭 K 項だけ記憶）が候補だが、上記より利得は限定的。
**派生課題（別途）**：`sqrt_cf_period` の O(周期²) を `HashMap` で状態→添字を引く O(周期) に改善できる
（周期そのものを使う他用途のための単体最適化として有意）。

**この測定の資産**：`sqrt_period_length_growth_measurement`（決定論的・高速）が周期増大を回帰的に記録する。

## ③ 実二次体 ℚ(√d) 代数層 — 最も深い拡張

**狙い**：`√2 + 1/2` のような算術結果は現在 Gosper ノード（比較は予算付きストリーム）。
しかし `(a + b√d)/c` 形は同じ体で +−×÷ が閉じ、**比較も O(1) で厳密決定**（符号判定→平方）。
この層を持てば SQRT 由来の広いクラスが「遅延・半決定」から「即時・全決定（U が出ない）」へ。

**論点**：

- 表現は内部的（`AlgebraicSqrt` 同様、値の意味・表示・直列化は不変）に留める。観測可能性を
  変えないことが必須（§4.2.2「どの表現かは観測不能」）。
- 体の混在（√2 と √3 → ℚ(√2,√3) は次数4）は係数が増える。一般化は代数的数
  （最小多項式＋分離区間）で、これは大きめのプロジェクト。まず単一二次体に限定した薄い層から。
- 演算の構築（`add`/`mul`/`div`）で「両オペランドが同じ √d 体に属するか」を判定し、
  属すれば代数層へ、さもなくば現行 Gosper へフォールバック。

**検証**：代数層の比較・演算結果が、現行 Gosper 経路の展開と項列・値で完全一致することを
差分テストで固定。① と同様、代数層が短絡した比較の割合を指標化。

**到達範囲の測定（`quadratic_field_reach_measurement`、決定論的）**：値の式木が参照する
radicand の平方因子除去部（field key）の異なり数で、Gosper（算術）結果を分類した。

| コーパス | single-field | multi-field | 同一単一体ペア |
| --- | --- | --- | --- |
| 単一体（有理数＋ℚ(√2) のみ：2,8,18,32,50） | **100%** | 0% | **100%** |
| 混在（多数の異なる体の surd） | 0% | **100%** | 0% |

**結論（射程は all-or-nothing、体局所性で決まる）**：

- 実二次体は四則で**閉じる**（測定で確認：`m1 == 0` を assert）。よって**単一の体に閉じた計算**では
  すべての算術結果が単一体に留まり、③は全面適用（全比較 O(1)、U が出ない）。
- 逆に**一度でも異なる体を跨ぐ演算**を行うと即座に高次体（例 ℚ(√2,√3) は次数4）へ脱出し、
  単一二次体③層は不適用になる（混在コーパスで single=0%）。
- したがって③の利得は算術そのものではなく**ワークロードの体局所性**に支配される。これは
  合成データからは決められない利用ドメインの問題であり、③を「単一体なら高速路・それ以外は
  現行 Gosper へフォールバック」という**機会主義的最適化**（決して悪化させない）として入れる価値はあるが、
  実装規模が大きいため、単一体 workload の実需が確認できてから着手するのが妥当。

## ④ NTT 整数乗算（「虚数を消す」発想）— 測定により見送り

**着想**：量子力学の「虚数不要」（複素構造を実線形代数で表す）の作動核を数値処理に写すと、
**FFT 乗算の複素単位根を整数の単位根で置き換えた NTT（数論変換）**になる。NTT は虚数も浮動小数の
丸めも使わない厳密 Θ(n log n) 乗算で、Ajisai の「厳密・丸めなし」と理念的に合致する。Gosper 係数や
高精度 √ 展開で BigInt が巨大化する領域の乗算を速められるか、を検証した。

**測定**（`ntt_operand_reach_measurement` ＝リーチ、`bench_bigint_multiply_cost_curve` ＝コスト曲線）：

| 経路 | オペランドのビット規模 |
| --- | --- |
| √ 展開の convergent | ≈ 1.27 bits/項。比較予算256項 → **325 bit**、20000項でようやく 25,430 bit |
| `best_rational_approximation` 分母 10^5000 | ≈ **16,610 bit**（数千桁精度を明示要求した時） |
| 深い混在 Gosper 木（√演算2000回） | **最大係数 = 1 bit**（係数は 0/±1 のまま。増えるのは深さだけ） |

| 乗算オペランド | per_mul（debug, スケーリング比較用） |
| --- | --- |
| 256 bit | 1.4 µs |
| 4,096 bit | 74.7 µs |
| 65,536 bit | 4.77 ms |
| 1,048,576 bit | 311 ms |

→ 4倍サイズで時間 ≈7.6–8.6倍（指数 **≈1.5**）。`num-bigint` は 1M bit でも Karatsuba/Toom のままで、
**FFT/NTT を使っていない**（NTT なら大領域で漸近的に勝てる）。

**見送りの理由（前提が否定された）**：

- **通常運用のオペランドは ≤ 325–1000 bit**——NTT が Toom を抜く帯（経験的に数万〜十数万 bit）の
  **約100倍下**。利得ゼロ。
- 巨大整数は **`COMPARE-WITHIN` 予算 ≳2万 / `best_rational_approximation` 分母 ≳10^5000** のような
  **明示的高精度要求でのみ**発生する稀な opt-in。
- 純 √ 演算は係数 magnitude が膨張せず（0/±1）、増えるのは**深さ**。よって「巨大整数乗算」自体が
  通常は現れない。
- 仮にその稀な領域を速くしたくなっても、自前 NTT より **GMP/FFT 実装を持つ BigInt バックエンド
  （`rug` 等）への差し替え**の方が低リスク・即効。

→ **カスタム NTT は実装しない。** 知的には魅力的だが、この処理系の実オペランドサイズでは出番がない
（②周期キャッシュと同型の「測定が無駄実装を防いだ」事例）。資産として
`ntt_operand_reach_measurement`（リーチを回帰記録）と `bench_bigint_multiply_cost_curve`
（`--ignored`、乗算コスト曲線）を残す。
