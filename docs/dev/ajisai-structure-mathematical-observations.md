# Ajisai 構造の数学的観察ノート

> 非正準・観察記録。Ajisai の意味論は `SPECIFICATION.html` のみが定義する。
> 本ノートは実装（主に `rust/src/types/`）を読んで気づいた数学的構造と、
> 独立に計算で確かめた事実をまとめた覚え書きである。設計判断や互換性方針を
> 定めるものではない。

## 0. 要約

Ajisai の数値モデルは「実数 = 部分商の遅延ストリーム」という**余代数的**な表現に立脚し、
算術は Gosper の双一次変換（2×2 / 2×2×2 整数係数）として実装されている。この構造を
観察すると、いくつかの綺麗な性質と、ひとつ「兆し」と呼べる反例が見つかる。中心的な発見は次の一点：

> **NICF（最近整数連分数）による比較の高速化は、同一予算で見たとき
> 正則連分数（RCF）を*点ごとには*支配しない。** 近接する有理数ペアの約 2.8% で、
> RCF の方が少ない項数で順序を決定する反例が存在する。原因は「正則 CF の `…, a, 1` という
> 1-折りたたみ非一意性」を NICF が正準化することで、RCF が露わにした分岐が 1 項ぶん
> 繰り延べられる場合があるため。順序の*値*は決して変わらない（決定したときは必ず一致する）が、
> U→決定の境界は必ずしも有利側にだけ動くわけではない。

仕様 §4.2.5 / §7.4.1.1 の「NICF は決定する頻度を減らさない／境界を有利な方向にだけ動かす」は、
平均的・意図的性質としては妥当だが、**項数を直接突き合わせた普遍命題としては反例を持つ**。
仕様自身が「NICF 項数と RCF 項数の対応は観測不能で依拠してはならない」と但し書きしている点が、
この反例に対する防御線になっている。一方、対応する Rust テスト
(`nicf_decides_at_least_as_often_as_rcf`) は手選びの 4 ペアしか確認しておらず、普遍性は保証しない。

## 1. 値モデル：実数の余代数的（ストリーム）表現

`rust/src/types/continued_fraction.rs` の `ExactReal` は実数を 3 つの形で持つ：

- `Rational(Fraction)` — 有限 CF（既約分数）。
- `AlgebraicSqrt { radicand }` — 非負有理数の √。CF は**最終的に周期的**（Lagrange の定理）で遅延生成。
- `Gosper(Arc<Gosper>)` — 1～2 オペランドの未評価双一次変換
  \((a xy + b x + c y + d)/(e xy + f x + g y + h)\)。

観察できる構造：

1. **算術 = 行列/テンソルの作用。** 加減乗除はすべて Gosper の Möbius（単項、2×2）あるいは
   bihomographic（二項、2×2×2）変換に帰着する（`bihom_apply_add/sub/mul/div`）。実数体の演算が
   「整数係数の小さな線形作用素を CF ストリームに合成していく」操作に一様化されている。
   有理数オペランドは `Fraction` の高速路へ折られ、Gosper ノードを作らない（`mobius_apply` の早期短絡）。

2. **O(1) で読める量がある。** canonical CF を \([a_0; a_1, a_2, \dots]\) とすると
   - `floor` = \(a_0\)（部分商を 1 つ引くだけ。無理数でも O(1)）、
   - `ceil` = \(a_0\) または \(a_0+1\)、
   - `round` は高々 3 項。
   これは「小数表現なら無限桁を見ないと決まらない床関数が、CF では先頭 1 項で確定する」という、
   表現選択がもたらす計算量上の非自明な利得である（`floor`/`ceil`/`round` の doc コメントに証明が書かれている）。

3. **最良有理近似 = convergent。** `best_rational_approximation` は標準の漸化式
   \(h_k = a_k h_{k-1} + h_{k-2},\ k_k = a_k k_{k-1} + k_{k-2}\)
   を分母上限まで回す。CF の convergent が最良近似であるという古典定理（Lagrange/Hurwitz）の直接実装。

4. **Lagrange 周期性の利用。** `sqrt_cf_period` は二次無理数 √D の状態 \((P,Q)\) を循環するまで辿り、
   `(pre_period, period)` を返す。無限展開を有限記述に畳み、N 番目の部分商を周期へのインデックスで O(1) に読む。

## 2. 等号・順序の半決定性と「決定の島」

CF が一致すれば値が等しい（仕様 §4.2.1）。逆に、相異なる実数は有限の項で必ず分岐する。
しかし**等しい無理数同士**は永遠に分岐しないので、純粋に CF ストリームを回すだけでは
等号判定は予算切れ＝`U`（Unknown, 停滞）に落ちる。これが「比較は半決定的」という本質。

Ajisai はここに**代数的短絡（決定の島）**を置く（`cmp_with_budget_tracked`）：

- `√r` vs `√s` → √ の単調性により `r` vs `s` で O(1) 決定（等しい無理数の √ も `Equal` を即決）。
- `√r` vs 有理数 `q` → `cmp_sqrt_vs_rational`（`q ≤ 0` なら即 `Greater`、さもなくば `r` vs `q²`）。

つまり「原理的には半決定的な等号」の中に、代数的構造から**完全決定できる島**を切り出している。
それ以外（Gosper 経由の和・積など）は予算付きストリーム比較に委ね、決まらなければ `U` を返し、
一致した先頭項数を `agreedPrefix` として診断に残す。これは「決定不能を値の不在(NIL)と混同せず、
論理的未決定として可視化する」という言語全体の方針（水のメタファ：Bubble vs Stagnation）の
数値版になっている。

## 3. 中心的観察：NICF 高速化は点ごとには RCF を支配しない

### 3.1 仕様・実装が主張していること

比較の予算は**最近整数連分数（NICF, semiregular CF）**の項で数える（§4.2.5, §7.4.1.1）。
NICF は収束が速いため、同じ予算でより少ない項で順序が露わになる、というのが設計意図。
実装 `NicfStream` は丸めの tie-break を **round-half-down**（剰余を \((-1/2, 1/2]\) に取る、
`ceil((2n-d)/(2d))`）で行う。仕様の文言は

> NICF は決定した順序を覆さない。U→決定の境界を**有利な方向にだけ**動かす。

であり、Rust テスト `nicf_decides_at_least_as_often_as_rcf` はこれを 4 つの手選びペアで確認している。

### 3.2 独立検証で見つかった反例

`scratchpad/cf_probe.py`（標準ライブラリ `fractions` のみ）で、実装の `round_half_down` と
同一式の NICF と、floor による RCF をそれぞれ展開し、**最初に分岐する項のインデックス**
（= その予算で順序が決定する境界）を比較した。近接有理数ペア 20,000 件：

| 観察 | 割合 |
| --- | --- |
| NICF が**早く**分岐（決定が速い） | 82.0% |
| 同点 | 15.1% |
| **RCF が早く分岐（NICF が遅い）** | **2.8%** |

平均決定項数は RCF 5.72 → NICF 4.04（約 1.42 倍速）で、意図どおり概ね速い。
**しかし RCF が勝つ 2.8% は普遍的支配の反例**である。具体例（`a`,`b` は隣接有理数）：

```
a=-9/10  b=-364/405
   RCF(a)=[-1, 10]            RCF(b)=[-1, 9, 1, 7, 5]    -> 項1で分岐
  NICF(a)=[-1, 10]           NICF(b)=[-1, 10, -8, -5]    -> 項2で分岐
```

RCF は項1（`10` vs `9`）で分岐＝決定するのに、NICF は先頭2項 `[-1, 10]` が一致してしまい、
分岐が項2へ繰り延べられる。よって「予算 = 2 NICF項」のような点では、RCF 単位なら決まっていた
順序が NICF 単位では `U` になりうる。

### 3.3 構造的説明：1-折りたたみの正準化が代償になる

正則 CF には末尾 `…, a` と `…, a-1, 1` という 2 通りの表現があり、NICF（最近整数化）は
`…, x, 1, …` を「次項が 1 ⇔ 端数 > 1/2 ⇔ 切り上げ」で `…, x+1, …` に**畳み込む**。
上例では `b` の RCF `[-1, 9, 1, …]` の `9,1` が NICF で `10` に畳まれ、`a` の `[-1,10]` と
先頭がそろってしまう。**RCF が「9 か 10 か」で晒していた差を、NICF の正準化が一時的に隠す。**
これが「NICF が遅れる」ケースの正体で、`…, a, 1`（次項=1）パターンを含む分岐の近傍で系統的に生じる。

### 3.4 含意（バグではない／但し精密化が要る命題）

- **正しさは保たれる。** NICF が決定したときの順序は RCF と必ず一致する（検証済み、反例は
  「決定の有無の境界」のみに現れ、決定された値の食い違いではない）。
- **観測可能量への影響はない。** 利用者は NICF 予算しか観測できず、仕様も「RCF↔NICF の項対応は
  観測不能で依拠してはならない」と明言する。よって COMPARE-WITHIN の契約は破れない。
- **精密化の余地。** 「同一予算で決定頻度を減らさない／境界を有利側にだけ動かす」は、項数を
  直接突き合わせる読み方では**普遍的には偽**。平均的性質か、あるいは「観測可能な NICF 予算の下で」
  という限定命題として述べるのが正確。テストも 4 ペアの存在確認に留まるため、普遍性の主張を
  支えてはいない。`scratchpad/cf_probe.py` の反例探索は、これを補強するランダムテスト
  （NICF が決定したら順序が RCF と一致することを property として確認しつつ、
  「NICF 早分岐率」を回帰指標として観測）に発展させられる。

## 4. その他の小さな観察

- **三値論理と三分割の対応。** Bubble(`NIL`, 不在) / Stagnation(`UNKNOWN`, 未決定) / Error(チャネル誤り) の
  3 分割は、数値側では「NIL = 不在」「U = 予算内で順序未決定」「err = 形が不正」に対応し、
  論理側の強 Kleene 三値（K3）と整合している。比較の `U` が論理 `U` へ自然に流れ込む設計。
- **`SemanticKind` と `ValueShape` がともに 8 値。** `value_axes.rs` の意味種別
  (Number/Collection/Record/Code/Process/Supervisor/Absence/Unknown) と
  形状 (Scalar/Vector/Tensor/Record/CodeBlock/Handle/Absence/Unknown) はいずれも 8 要素で、
  末尾 2 つ (Absence/Unknown) を両軸が共有する。「不在・未知」が直交軸の交点として一貫配置されている。
- **区間演算は別格子。** `interval.rs` は有理端点の区間 [lo,hi] による外向き丸めを持ち、CF の厳密路とは別系統の
  近似格子（√ の二分探索 `sqrt_rational_interval` など）として共存する。
- **Lévy 型の幾何成長。** convergent の分母は項数に対しほぼ幾何級数的に増える
  （`cf_probe.py` で確認、定数は有限有理数コーパスゆえ理論値 \(\pi^2/(12\ln 2)\) からは下振れ）。
  これは「N 項で約 \(e^{KN}\) の分解能」を意味し、比較予算が分解能の対数に対応することの根拠。
