# Ajisai Adoption Strategy: Music Programming Platform

## Executive Summary

Paul Grahamの「Being Popular」の最も重要な教訓：
> 「プログラミング言語は孤立しては存在しない。ハッカーはふつう何かをハックする」

Ajisaiを**音楽プログラミングプラットフォーム**として位置づけ、特にハンドパンをはじめとする**倍音楽器**の愛好家コミュニティから採用を広げる。

---

## Why Music Programming?

### Ajisaiの本質：Vector指向プログラミング

Ajisaiは「テンソル処理言語」ではなく、**Vector指向のスタック言語**である：

```ajisai
# Ajisaiの核心：異種データ混在可能なネスト可能Vector
[ 440 'ding' [ 3/2 5/4 ] TRUE ]

# スタック自体が暗黙の最外殻Vector
[ 440 ] [ 523 ] [ 659 ]   # スタック上に3つのVectorが積まれている
.. AUDIO                   # スタック全体（=暗黙のVector）に対して操作
```

**LISPのリスト構造に通ずる設計思想**：
- Vectorの中にVectorをネストして構造を表現（フラクタル構造）
- 異種データ混在可能：数値・文字列・真偽値・Vectorを自由に組み合わせ
- スタック=暗黙のVector、`..`モードでスタック全体を一つのVectorとして操作

### Ajisaiの強みと音楽の親和性

| Ajisaiの特徴 | 音楽への応用 |
|------------|-----------|
| **有理数演算** | 純正律の音程比（3/2, 4/3, 5/4）を正確に表現 |
| **Vector指向** | 音符・フレーズ・楽曲を自然にネスト構造で表現 |
| **異種データ混在** | `[ 440 'C4' 1/4 ]` のように周波数・音名・音価を一つのVectorに |
| **関数型スタイル** | MAP/FOLDでパターン生成・変形が簡潔 |
| **ブラウザ動作** | Web Audio APIで即座に音が出せる |
| **分数→和音** | 既存のDTMF機能が拡張可能 |

### なぜハンドパンか？

1. **ニッチだが熱心なコミュニティ** - 新しいツールに積極的
2. **倍音の美しさ** - 純正律・有理数との相性が抜群
3. **スケールベース** - 特定の音階で演奏する楽器
4. **視覚的にも魅力的** - デモ動画が映える
5. **作者の情熱** - 愛好者としての深い理解

## Current State

### 既存のオーディオ機能

```ajisai
# 単音再生
[ 440 ] AUDIO

# 和音（分数で表現）
[ 1209/697 ] AUDIO

# シーケンス
[ 440 494 523 587 659 ] AUDIO

# マルチトラック（スタック全体を同時再生）
[ 440 523 659 ] [ 349 440 523 ] .. AUDIO
```

### 課題

1. **周波数直接指定** - 音名（C4, D#3）で指定できない
2. **音色がシンプル** - 正弦波のみ、ハンドパンらしさがない
3. **スケール定義なし** - 毎回周波数を計算する必要がある
4. **リズム表現が弱い** - 固定長（0.5秒）のみ

## Proposed Enhancements

### Phase 1: 音名と音程（すぐに実装可能）

```ajisai
# 音名→周波数変換ワード
'C4' NOTE              # → [ 261.63 ]
'A4' NOTE              # → [ 440 ]
'D#3' NOTE             # → [ 155.56 ]

# 純正律の音程比（有理数の強みを活かす）
[ 440 ] [ 3/2 ] *      # → [ 660 ] （完全5度上）
[ 440 ] [ 5/4 ] *      # → [ 550 ] （長3度上）

# スケール定義
[ 'D3' 'A3' 'Bb3' 'C4' 'D4' 'E4' 'F4' 'A4' ] 'KURD' DEF
KURD 'NOTE' MAP        # → 全ノートを周波数に変換
```

### Phase 2: ハンドパンスケールプリセット

```ajisai
# 組み込みスケール
SCALE_KURD             # D Kurd (最もポピュラー)
SCALE_CELTIC           # Celtic Minor
SCALE_AMARA            # D Amara
SCALE_INTEGRAL         # Integral

# スケールの使用例
SCALE_KURD             # → [ 146.83 220 233.08 261.63 ... ]
[ 0 2 4 5 ] GET AUDIO  # → スケールの特定のノートを再生
```

### Phase 3: 音色と倍音

```ajisai
# 波形タイプ指定
[ 440 ] 'sine' SYNTH      # 正弦波（デフォルト）
[ 440 ] 'triangle' SYNTH  # 三角波
[ 440 ] 'handpan' SYNTH   # ハンドパン風倍音合成

# 倍音構成の直接指定
[ 440 ] [ 1 0.5 0.3 0.1 ] HARMONICS
# → 基音 + 2倍音(50%) + 3倍音(30%) + 4倍音(10%)
```

### Phase 4: リズムとパターン

```ajisai
# 音価指定
[ 440 1/4 ] [ 523 1/8 ] [ 659 1/2 ] RHYTHM
# → 4分音符、8分音符、2分音符

# パターン生成
[ 0 2 4 ] 'STEP' MAP   # → スケール度数でパターン
[ 1 0 1 1 0 1 1 0 ] 'BEAT' MAP  # → リズムパターン

# ループ
[ 440 523 659 ] 4 LOOP AUDIO  # → 4回繰り返し
```

## Key Messages

### Elevator Pitch

> **Ajisai**: ブラウザで動く、有理数ベースの音楽プログラミング言語。
> 純正律の美しい和音を、分数でそのまま表現できる。

### Technical Differentiators

1. **正確な音程** - 浮動小数点誤差なしで純正律を表現
2. **簡潔な記法** - `[ 440 ] [ 3/2 ] *` で完全5度
3. **即座に音が出る** - インストール不要、ブラウザで完結
4. **Vector指向** - 音楽構造をネストしたVectorで自然に表現
