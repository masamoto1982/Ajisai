Data Model & Two-layer Architecture

Ajisai separates computation from presentation through a two-layer architecture. The Data Plane holds pure Fraction tensors — every value in the language is ultimately a fraction with an arbitrary-precision BigInt numerator and denominator. All arithmetic, comparison, and logic operations happen entirely within this plane. The Semantic Plane holds display metadata called DisplayHint, stored in a separate SemanticRegistry and queried only at output boundaries (PRINT, STR, GUI rendering). Computation never touches hints; hints never alter data.

ValueData — The Core Data Representation

Every value in Ajisai is represented by the ValueData enum:

pub enum ValueData {
    Scalar(Fraction),           // Single fraction
    Vector(Rc<Vec<Value>>),     // Nested vector
    Record { pairs, index },    // Key-value pairs
    Nil,                        // Absence of value
    CodeBlock(Vec<Token>),      // Deferred code
}

The Value struct contains only data: ValueData — no display hints or metadata live in the data plane.

Fraction (Number)

All numbers are arbitrary-precision fractions with a BigInt numerator and denominator. Fractions are auto-reduced via GCD on every operation. The sign is carried by the numerator; the denominator is always positive.

Formats

# Integer
42
-17

# Decimal (auto-converted to exact fraction)
3.14       # Stored as 157/50
0.5        # Stored as 1/2

# Fraction literal
1/3
22/7

# Scientific notation
1.5e2      # Stored as 150/1

Exact Arithmetic

Because all values are exact fractions, there are no rounding errors:

[ 1/3 ] [ 1/6 ] +    # Result: [ 1/2 ] (exact)
[ 1/10 ] [ 2/10 ] +  # Result: [ 3/10 ] (exact, no float drift)

# Rounding functions
[ 7/3 ] FLOOR    # Result: [ 2 ]
[ 7/3 ] CEIL     # Result: [ 3 ]
[ 5/2 ] ROUND    # Result: [ 3 ]

String

Strings are vectors of Unicode code points with a String display hint applied in the Semantic Plane. Internally, 'Hello' is a Vector of scalars [72, 101, 108, 108, 111]. The display hint tells the output layer to render it as text rather than a list of numbers.

'hello'
'日本語'          # Full Unicode support

# CHARS splits a string into individual characters
[ 'hello' ] CHARS     # Result: [ 'h' 'e' 'l' 'l' 'o' ]

# JOIN merges characters back into a string
[ 'h' 'e' 'l' 'l' 'o' ] JOIN    # Result: [ 'hello' ]

# CHR converts a code point number to a character
[ 65 ] CHR    # Result: [ 'A' ]

Boolean

Booleans are Scalar(1/1) for TRUE and Scalar(0/1) for FALSE, with a Boolean display hint. Since they are ordinary fractions, they participate in arithmetic normally (1 and 0).

TRUE
FALSE

# Comparisons produce booleans
[ 3 ] [ 5 ] <      # Result: [ TRUE ]
[ 3 ] [ 3 ] =      # Result: [ TRUE ]

# Logic operations
[ TRUE ] [ FALSE ] AND    # Result: [ FALSE ]
[ TRUE ] [ FALSE ] OR     # Result: [ TRUE ]
[ TRUE ] NOT              # Result: [ FALSE ]

Bracket Display

On input, all bracket types ([], {}, ()) are interchangeable. On display, brackets are auto-converted to match the depth-cycling convention:

[ 1 2 3 ]                    # Display: { 1 2 3 }
[ [ 1 2 ] [ 3 4 ] ]          # Display: { ( 1 2 ) ( 3 4 ) }

Rectangular Constraint

Elements at the same nesting dimension must have the same shape. A vector like [ [ 1 2 ] [ 3 ] ] is invalid because the inner vectors have different lengths.

Shape Operations

[ [ 1 2 3 ] [ 4 5 6 ] ] SHAPE      # Result: [ 2 3 ]
[ [ 1 2 3 ] [ 4 5 6 ] ] RANK       # Result: [ 2 ]
[ 1 2 3 4 5 6 ] [ 2 3 ] RESHAPE    # Result: [ [ 1 2 3 ] [ 4 5 6 ] ]
[ [ 1 2 ] [ 3 4 ] ] TRANSPOSE      # Result: [ [ 1 3 ] [ 2 4 ] ]

DisplayHint (Semantic Plane)

Display hints determine how a value is rendered to the user. They live in the SemanticRegistry, completely separate from the Value struct. Computation never reads or modifies hints.

HintRendering
AutoDefault — rendered as a number
NumberExplicit numeric display
StringCode points rendered as text
Boolean1/0 rendered as TRUE/FALSE
DateTimeTimestamp rendered as date/time
NilRendered as NIL

The words STR, NUM, and BOOL change the display hint of a value without altering the underlying fraction data.

NIL

NIL is a distinct ValueData::Nil variant — it is not a fraction, not zero, not an empty collection. Empty brackets [ ] are errors, not NIL.

Broadcasting

Ajisai uses NumPy/APL-style broadcasting for arithmetic on values with different shapes. The rules are:

  1. Compare shapes from right to left (trailing dimensions first).
  2. Each dimension is compatible if both sizes are equal, or one of them is 1.
  3. Missing dimensions are padded with 1 on the left.
# Scalar broadcasts across a vector
[ 5 ] [ 1 2 3 ] +    # Result: [ 6 7 8 ]

# Vector broadcasts across rows of a matrix
[ [ 1 2 3 ] [ 4 5 6 ] ] [ 10 20 30 ] +
# Result: [ [ 11 22 33 ] [ 14 25 36 ] ]

# Scalar broadcasts across a matrix
[ [ 1 2 ] [ 3 4 ] ] [ 10 ] +
# Result: [ [ 11 12 ] [ 13 14 ] ]
