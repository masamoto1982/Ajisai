Data Types & FlowTokens

In Ajisai's Fractional Dataflow model, every value on the stream is a FlowToken — a typed fraction that is consumed exactly once by an operation and produces a result plus a remainder. The types below describe the kinds of FlowTokens that flow through the consume/remainder chain.

Number (Fraction FlowToken)

All numbers are internally fractions — no rounding errors. This conservation-preserving representation ensures that no value is lost as tokens flow through operations:

# Integers
42
-17

# Decimals (auto-converted to fractions)
3.14       # Stored as 157/50
0.5        # Stored as 1/2

# Fractions
1/3
22/7

# Arithmetic
[ 1/3 ] [ 1/6 ] +    # Result: [ 1/2 ] (exact)

Rounding Functions

[ 7/3 ] FLOOR    # Result: [ 2 ]
[ 7/3 ] CEIL     # Result: [ 3 ]
[ 5/2 ] ROUND    # Result: [ 3 ]

String (Text FlowToken)

'hello'
'日本語'          # UTF-8 supported

# Split into characters
[ 'hello' ] CHARS     # Result: [ 'h' 'e' 'l' 'l' 'o' ]

# Join characters
[ 'h' 'e' 'l' 'l' 'o' ] JOIN    # Result: [ 'hello' ]

Boolean (Logic FlowToken)

TRUE
FALSE

# Comparisons
[ 3 ] [ 5 ] <      # Result: [ TRUE ]
[ 3 ] [ 3 ] =      # Result: [ TRUE ]

# Logic
[ TRUE ] [ FALSE ] AND    # Result: [ FALSE ]
[ TRUE ] [ FALSE ] OR     # Result: [ TRUE ]
[ TRUE ] NOT              # Result: [ FALSE ]

Vector (Composite FlowToken)

Vectors are composite FlowTokens that carry multiple values through the fraction flow. Maximum 4 dimensions supported:

DimExample
1D[ 1 2 3 ]
2D[ [ 1 2 ] [ 3 4 ] ]
3D[ [ [ 1 2 ] [ 3 4 ] ] ]
4D[ [ [ [ 1 ] ] ] ]

Shape Operations

[ [ 1 2 3 ] [ 4 5 6 ] ] SHAPE      # Result: [ 2 3 ]
[ [ 1 2 3 ] [ 4 5 6 ] ] RANK       # Result: [ 2 ]
[ 1 2 3 4 5 6 ] [ 2 3 ] RESHAPE    # Result: [ [ 1 2 3 ] [ 4 5 6 ] ]
[ [ 1 2 ] [ 3 4 ] ] TRANSPOSE      # Result: [ [ 1 3 ] [ 2 4 ] ]

DateTime (Temporal FlowToken)

# Current time
NOW    # Result: [ @2024-11-25 14:00:00 ]

# Convert to components
NOW 'LOCAL' DATETIME    # Result: [ [ 2024 11 25 14 0 0 ] ]

# Create from components
[ [ 2024 12 25 0 0 0 ] ] 'LOCAL' TIMESTAMP

Type Conversion (FlowToken Transform)

WordExample
STR[ 42 ] STR → [ '42' ]
NUM[ '42' ] NUM → [ 42 ]
BOOL[ 1 ] BOOL → [ TRUE ]
CHARS[ 'abc' ] CHARS → [ 'a' 'b' 'c' ]
JOIN[ 'a' 'b' ] JOIN → [ 'ab' ]

Broadcasting (Fraction Flow Expansion)

Broadcasting extends a FlowToken across dimensions via remainder passing — like a TPU distributing a single value across parallel lanes with no materialized intermediates:

# Scalar broadcast — single fraction flows across the vector
[ 5 ] [ 1 2 3 ] +    # Result: [ 6 7 8 ]

# Row broadcast — fraction flow expands along rows
[ [ 1 2 3 ] [ 4 5 6 ] ] [ 10 20 30 ] +
# Result: [ [ 11 22 33 ] [ 14 25 36 ] ]
