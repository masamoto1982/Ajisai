Design Philosophy

Manifesto: Ajisai is a Fractional Dataflow language. Every value flows through operations as a fraction — consumed and producing a remainder, like a TPU pipeline with no materialized intermediates. Each token is spent exactly once, like a UTXO: what enters must exit as result + remainder, preserving the conservation law.

The Name "Ajisai"

The name "Ajisai" (紫陽花, hydrangea) metaphorically represents how small words come together to form functionality, like how small flowers form a hydrangea cluster — each petal a fraction flowing into the whole.

Note: What appears to be petals on a hydrangea are actually sepals.

Core Design Principles

1. Fractional Dataflow Computation

Ajisai implements a Fractional Dataflow paradigm (FORTH-inherited postfix notation) with Reverse Polish Notation (RPN):

  • Eliminates the need for parentheses to specify operation order
  • Makes fraction flow explicit and visible — every value enters, is consumed, and produces a remainder
  • Enables simple composition of operations via remainder passing
  • TPU analogy: Data streams through operations with no materialized intermediates — each word consumes its input and forwards the remainder
  • UTXO analogy: Each FlowToken is spent exactly once; the output is result + remainder, enforcing the conservation law

2. Vector-centric Data Model

All container data is represented as nestable Vectors (similar to LISP's list structure), serving as fraction flow carriers:

  • Bracket [ ] nesting expresses dimensions
  • Supports tensor-like operations (SHAPE, RESHAPE, TRANSPOSE)
  • Allows heterogeneous data: [ 1 'hello' TRUE [ 2 3 ] ]
  • NumPy/APL-style broadcasting across fraction flows

3. Dimension Limit (Maximum 4D)

Based on human cognitive limits, Ajisai restricts dimensions to 4:

DimAxesExample
1Dcol[ 1 2 3 ]
2Drow, col[ [ 1 2 ] [ 3 4 ] ]
3Dlayer, row, col[ [ [ 1 2 ] ] ]
4Dtime, layer, row, col[ [ [ [ 1 ] ] ] ]

4. Conservation-Preserving Fraction Arithmetic

All numbers are internally represented as fractions, eliminating rounding errors and enforcing the conservation law:

  • 5 stored as Fraction(5, 1)
  • 0.5 stored as Fraction(1, 2)
  • Capable of handling extremely large numbers
  • Every arithmetic operation preserves exact quantity — nothing is lost to rounding, upholding the Fractional Dataflow conservation guarantee

5. "No Change is an Error" Principle

Operations that result in no change produce an error — in the Fractional Dataflow model, a no-op means the consume/remainder chain was not honored:

  • Prevents silent no-ops that may hide bugs
  • Examples: [ 1 ] REVERSE, [ 1 2 3 ] SORT (already sorted)

6. Built-in Word Protection

Built-in words cannot be deleted or overwritten, ensuring consistent remainder forwarding behavior.

The Consume/Remainder Model

Every operation in Ajisai follows the consume/remainder pattern:

  1. Consume: The operation takes its input FlowTokens from the stream
  2. Transform: The operation produces a result (no materialized intermediates, like a TPU)
  3. Remainder passing: Unconsumed values are forwarded as the remainder to the next operation

This is analogous to the UTXO (Unspent Transaction Output) model: each value-token is spent exactly once, and the "change" is passed forward. The conservation law ensures that no value is created or destroyed — only transformed.

Influences

  • FORTH: Postfix notation, word definitions (inherited as the syntactic foundation)
  • TPU / Streaming architectures: No materialized intermediates; data flows through operations
  • UTXO (Bitcoin): Consume/remainder model; each token spent exactly once
  • LISP: Nested list structures
  • APL/J: Array operations, broadcasting
  • NumPy: Broadcasting rules, shape operations
