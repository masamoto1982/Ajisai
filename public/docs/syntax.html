Syntax

Postfix Notation (RPN)

In Ajisai, operators follow their operands. This is postfix notation, also known as Reverse Polish Notation:

# Traditional infix: (1 + 2) * 3
# Ajisai postfix:
[ 1 ] [ 2 ] + [ 3 ] *    # Result: [ 9 ]

# No parentheses needed for precedence
[ 10 ] [ 3 ] - [ 2 ] *   # Result: [ 14 ]

Each word reads its operands from the values that precede it, executes, and pushes its result. Evaluation proceeds left to right with no ambiguity about operator precedence.

Basic Elements

Comments

Comments begin with # and extend to the end of the line:

# This is a comment
[ 1 2 3 ]  # Inline comment

Numbers

All numbers are stored as exact fractions internally:

42        # Integer
3.14      # Decimal (stored as 157/50)
1/3       # Fraction literal
-5        # Negative
1.5e2     # Scientific notation (stored as 150/1)

Strings

Strings are enclosed in single quotes and support full Unicode:

'hello'
'日本語'

Booleans

TRUE and FALSE are uppercase keywords:

TRUE
FALSE

NIL

NIL represents the absence of a value. It must be uppercase:

NIL

Vectors

Vectors are created with brackets. All three bracket types ([], {}, ()) are interchangeable on input:

# These are all equivalent on input
[ 1 2 3 ]
{ 1 2 3 }
( 1 2 3 )

# Nested vectors
[ [ 1 2 ] [ 3 4 ] ]

# Mixed value types
[ 1 'hello' TRUE ]

On display, Ajisai auto-converts brackets based on depth-cycling: dimension 1 uses {}, dimension 2 uses (), dimension 3 uses [], then the cycle repeats. Vectors can nest up to 10 dimensions.

Empty brackets [ ] are errors. Use NIL to represent absence.

Defining Words

Custom words are defined with a code block and DEF. Code blocks use { ... } or ( ... ) syntax:

# Define a word that doubles its input
{ [ 2 ] * } 'DOUBLE' DEF

# Use it
[ 5 ] DOUBLE    # Result: [ 10 ]

A word can include a description string as a third argument:

# Define with description
{ [ 2 ] MOD [ 0 ] = } 'IS-EVEN' 'Checks if even' DEF

# Use it
[ 4 ] IS-EVEN    # Result: [ TRUE ]
[ 7 ] IS-EVEN    # Result: [ FALSE ]

Custom word call chains are limited to 4 steps (main context + 4 = 5 total hierarchy levels).

Modifiers

Modifiers alter how the next word executes. They are order-independent and auto-reset after the word completes:

ModifierNameEffect
.Stack top modeDefault: operate on the top element of the stack
..Stack modeOperate on the entire stack as a single vector
,Consume modeDefault: consume operands after use
,,Bifurcation modeSplit flow mass: keep operands and push result
~Safe modeErrors become NIL instead of halting execution
!Force flagAllow DEL/DEF of words that have dependents
==PipelineVisual no-op marker for readability
=>Nil coalescingProvide a fallback value if the preceding value is NIL

Examples

# Stack mode: operate on entire stack
[ 1 ] [ 2 ] [ 3 ] .. LENGTH    # Result: [ 3 ]

# Bifurcation: keep operands + push result
[ 5 ] ,, [ 3 ] +    # Stack: [ 5 ] [ 8 ]

# Safe mode: errors become NIL
[ 'abc' ] ~ NUM    # Result: NIL

# Force: redefine a word with dependents
{ [ 3 ] * } 'TRIPLE' ! DEF

Pipeline Operator (==)

The pipeline operator is a visual no-op. It does nothing at runtime but makes multi-step transformations easier to read:

[ 1 2 3 4 5 ]
  == { [ 2 ] * } MAP           # Double each element
  == { [ 5 ] < NOT } FILTER    # Keep values that are not less than 5
  == [ 0 ] { + } FOLD          # Sum the results

Nil Coalescing (=>)

The nil coalescing operator provides a fallback value when the preceding value is NIL. If the value is not NIL, it passes through unchanged:

NIL => [ 0 ]       # Result: [ 0 ]   (NIL replaced by fallback)
[ 42 ] => [ 0 ]    # Result: [ 42 ]  (non-NIL passes through)

# Chained with safe mode for defensive defaults
[ 'abc' ] ~ NUM => [ 0 ]    # Result: [ 0 ]
