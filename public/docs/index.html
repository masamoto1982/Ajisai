Ajisai — Vector-oriented Fractional Dataflow Language

Manifesto: Data flows like water. In Ajisai, every value is a fraction that streams through operations — each word consumes a portion and passes the remainder downstream. The DisplayHint is merely the surface ripple, not the substance beneath. What you see at the output boundary is interpretation; what flows through the pipeline is pure fractional data, conserved without loss.

What is Ajisai?

Ajisai is a vector-oriented, fractional-dataflow programming language. It inherits postfix (Reverse Polish) notation from FORTH, runs on a WebAssembly runtime compiled from Rust, and provides an interactive web-based GUI.

At its core, Ajisai employs a two-layer architecture:

  • Data Plane — Pure Fraction tensors (Scalar, Vector, Record, Nil, CodeBlock). All computation happens here. Values are consumed and remainders forwarded, preserving the conservation law.
  • Semantic Plane — DisplayHint (Auto, Number, String, Boolean, DateTime, Nil) and ValueExt metadata live in the SemanticRegistry, queried only at output boundaries (PRINT, STR, GUI). Computation never touches this layer.

The name "Ajisai" (hydrangea in Japanese) represents how small words come together to form functionality, like how small flowers form a hydrangea cluster.

📚

Fractional Dataflow

Every value carries a FlowToken with consume/remainder tracking. Operations consume from remaining flow mass; the conservation law guarantees: initial_total = sum(consumed) + final_remainder.

🧮

Vector-oriented Processing

NumPy/APL-style shape broadcasting for arithmetic. Support for up to 10 dimensions, with bracket types [] {} () cycling every 3 levels for display.

🔢

Two-layer Architecture

Data Plane holds pure Fraction tensors for computation. Semantic Plane holds DisplayHint and metadata, queried only at output boundaries. Clean separation — no metadata in the data path.

🚀

WebAssembly Runtime

Rust core compiled to WASM with SIMD128 support for fast in-browser execution. Arbitrary-precision fraction arithmetic with no rounding errors.

Quick Example

# Vector creation
[ 1 2 3 ]               # 1D vector
[ [ 1 2 ] [ 3 4 ] ]     # 2D nested vector

# Broadcasting arithmetic
[ 5 ] [ 1 2 3 ] +       # Result: { 6 7 8 }

# User word definition
{ [ 2 ] * } 'DOUBLE' DEF
[ 5 ] DOUBLE             # Result: { 10 }

# Higher-order functions with pipeline
[ 1 2 3 4 5 ]
  == { [ 2 ] * } MAP           # Result: { 2 4 6 8 10 }
  == [ 0 ] { + } FOLD          # Sum: { 30 }
