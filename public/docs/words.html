Modifiers

.

Stack top mode (default). Operations affect only the top element of the stack.

[ 1 2 3 ] LENGTH # Operates on top element only

..

Stack mode — operates on the entire stack as a single sequence.

[ 1 ] [ 2 ] [ 3 ] .. REVERSE # Result: [ 3 ] [ 2 ] [ 1 ]

,

Consume mode (default). Operands are removed from the stack after the operation.

[ 1 ] [ 2 ] , + # Result: [ 3 ] (same as default behavior)

,,

Bifurcation mode — splits flow mass into child branches. Operands are retained on the stack and the result is pushed on top. This is flow mass splitting, not value copying.

[ 1 ] [ 2 ] ,, + # Result: [ 1 ] [ 2 ] [ 3 ]

~

Safe mode — errors become NIL instead of halting execution. Combine with => for default values.

[ 1 2 3 ] [ 10 ] ~ GET # Result: NIL [ 1 2 3 ] [ 10 ] ~ GET => [ 0 ] # Result: [ 0 ]

!

Force flag — allows DEL or DEF of words that have dependents.

! 'WORD' DEL # Force delete even if other words depend on it

Modifiers are order-independent: ,, .. and .. ,, produce the same result. Modifiers auto-reset after word execution.

Input Helpers

'

Single quote — starts a string literal.

'hello' # String value

FRAME

Create bracket structure with specified shape.

[ 2 3 ] FRAME # Creates a 2x3 frame structure

Position Operations (0-indexed)

GET Form

Get element at position. Negative index counts from end.

[ 10 20 30 ] [ 0 ] GET # Result: [ 10 ] [ 10 20 30 ] [ -1 ] GET # Result: [ 30 ]

INSERT Form

Insert element at position.

[ 1 3 ] [ 1 2 ] INSERT # Result: [ 1 2 3 ]

REPLACE Form

Replace element at position.

[ 1 2 3 ] [ 0 9 ] REPLACE # Result: [ 9 2 3 ]

REMOVE Form

Remove element at position.

[ 1 2 3 ] [ 1 ] REMOVE # Result: [ 1 3 ]

Quantity Operations

LENGTH Form

Get element count.

[ 1 2 3 4 5 ] LENGTH # Result: [ 5 ]

TAKE Form

Take N elements from start (positive) or end (negative).

[ 1 2 3 4 5 ] [ 3 ] TAKE # Result: [ 1 2 3 ] [ 1 2 3 4 5 ] [ -2 ] TAKE # Result: [ 4 5 ]

Vector Operations

SPLIT Form

Split vector at specified sizes.

[ 1 2 3 4 5 6 ] [ 2 3 ] SPLIT # Result: [ 1 2 ] [ 3 4 5 ] [ 6 ]

CONCAT Form

Concatenate two vectors.

[ 1 2 ] [ 3 4 ] CONCAT # Result: [ 1 2 3 4 ]

REVERSE Form

Reverse element order.

[ 1 2 3 ] REVERSE # Result: [ 3 2 1 ]

RANGE Form

Generate inclusive range.

[ 0 5 ] RANGE # Result: [ 0 1 2 3 4 5 ]

REORDER Form

Reorder elements by index vector.

[ 10 20 30 ] [ 2 0 1 ] REORDER # Result: [ 30 10 20 ]

SORT Form

Sort ascending.

[ 3 1 4 1 5 ] SORT # Result: [ 1 1 3 4 5 ]

COLLECT None

Collect N elements from the stack into a vector.

1 2 3 3 COLLECT # Result: [ 1 2 3 ]

String Operations

CHARS Map

Split string into character vector.

'hello' CHARS # Result: [ 'h' 'e' 'l' 'l' 'o' ]

JOIN Map

Join character vector into string.

[ 'h' 'e' 'l' 'l' 'o' ] JOIN # Result: 'hello'

Type Conversion

NUM Map

Convert to number. Returns NIL on failure.

'123' NUM # Result: [ 123 ] 'abc' NUM # Result: NIL

STR Map

Convert value to string representation.

[ 123 ] STR # Result: '123'

BOOL Map

Normalize value to boolean.

'true' BOOL # Result: TRUE [ 100 ] BOOL # Result: TRUE

CHR Map

Convert number to Unicode character.

[ 65 ] CHR # Result: 'A'

Arithmetic

+ - * / Fold

Element-wise arithmetic operations with broadcasting.

[ 1 2 3 ] [ 10 ] + # Result: [ 11 12 13 ] [ 1 2 3 ] [ 4 5 6 ] * # Result: [ 4 10 18 ] [ 10 ] [ 3 ] / # Result: [ 10/3 ]

MOD Fold

Modulo operation.

[ 7 ] [ 3 ] MOD # Result: [ 1 ]

FLOOR CEIL ROUND Map

Rounding operations.

[ 7/3 ] FLOOR # Result: [ 2 ] [ 7/3 ] CEIL # Result: [ 3 ] [ 5/2 ] ROUND # Result: [ 3 ]

Comparison

= < <= Fold

Comparison operators. There are no > or >= operators — use < and <= with swapped operands instead.

[ 1 2 ] [ 1 2 ] = # Result: TRUE [ 1 ] [ 2 ] < # Result: TRUE [ 2 ] [ 2 ] <= # Result: TRUE

Logic

AND OR Fold

Logical operations using three-valued Kleene logic with NIL.

TRUE FALSE AND # Result: FALSE TRUE FALSE OR # Result: TRUE TRUE NIL AND # Result: NIL FALSE NIL AND # Result: FALSE (absorbed)

NOT Map

Logical negation.

TRUE NOT # Result: FALSE FALSE NOT # Result: TRUE

Higher-Order Functions

MAP Form

Apply a word or code block to each element.

[ 1 2 3 ] 'DOUBLE' MAP # With word name [ 1 2 3 ] { [ 2 ] * } MAP # With code block

FILTER Form

Keep elements that satisfy a condition. Returns NIL if no elements match.

[ 1 2 3 4 5 6 ] { [ 2 ] MOD [ 0 ] = } FILTER # Result: [ 2 4 6 ]

FOLD Form

Reduce with an initial value and an accumulating operation.

[ 1 2 3 4 5 ] [ 0 ] { + } FOLD # Result: [ 15 ]

Shape Operations

SHAPE Map

Get the shape vector of a value.

[ [ 1 2 3 ] [ 4 5 6 ] ] SHAPE # Result: [ 2 3 ]

RANK Map

Get the rank (number of dimensions).

[ [ 1 2 3 ] [ 4 5 6 ] ] RANK # Result: [ 2 ]

RESHAPE Form

Reshape a value to given dimensions.

[ 1 2 3 4 5 6 ] [ 2 3 ] RESHAPE # Result: [ [ 1 2 3 ] [ 4 5 6 ] ]

TRANSPOSE Form

Transpose a matrix (swap rows and columns).

[ [ 1 2 ] [ 3 4 ] ] TRANSPOSE # Result: [ [ 1 3 ] [ 2 4 ] ]

FILL Form

Fill a shape with a value.

[ 3 3 ] [ 0 ] FILL # Result: 3x3 matrix of zeros

Constants

TRUE FALSE NIL

Boolean and null constants.

TRUE # Pushes TRUE (scalar 1 with Boolean hint) FALSE # Pushes FALSE (scalar 0 with Boolean hint) NIL # Pushes NIL (distinct variant, not a fraction)

IDLE

No-op word. Does nothing. Commonly used as the else guard in COND.

IDLE # Nothing happens

Control Flow

COND Form

Evaluate guard/body pairs and execute the first matching branch. Use IDLE as a catch-all guard.

[ 42 ] { [ 0 ] < } { 'negative' } { IDLE } { 'positive' } COND

== (Pipeline)

Visual data flow marker (no-op). Used to improve readability of pipelines.

[ 1 2 3 ] == { [ 2 ] * } MAP # Readable pipeline

=> (Nil Coalescing)

Provide a fallback value when the stack top is NIL. Non-NIL values pass through unchanged.

NIL => [ 0 ] # Result: [ 0 ] [ 42 ] => [ 0 ] # Result: [ 42 ]

DateTime

NOW

Get current Unix timestamp.

NOW # Result: [ 1732531200 ]

DATETIME

Convert timestamp to datetime vector.

[ 1732531200 ] 'LOCAL' DATETIME # Result: [ 2024 11 25 23 0 0 ]

TIMESTAMP

Convert datetime vector to timestamp.

[ 2024 11 25 23 0 0 ] 'LOCAL' TIMESTAMP # Result: [ 1732531200 ]

Metaprogramming

EXEC

Execute a vector as code.

{ [ 1 ] [ 2 ] + } EXEC # Result: [ 3 ]

EVAL

Parse and execute a string as code.

'[ 1 ] [ 2 ] +' EVAL # Result: [ 3 ]

HASH

Deterministic hash of any value.

'hello' HASH # Result: [ 0.xxx ] [ 128 ] 'hello' HASH # 128-bit hash output

CSPRNG

Cryptographically secure random number generation.

[ 6 ] [ 1 ] CSPRNG # Result: random number in [0, 5/6] [ 5 ] CSPRNG # Generate 5 random numbers

Word Management

DEF

Define a user word. Optionally include a description string.

{ [ 2 ] * } 'DOUBLE' DEF { [ 2 ] * } 'DOUBLE' 'Multiply by two' DEF

DEL

Delete a user word. Use ! to force delete words with dependents.

'DOUBLE' DEL ! 'DOUBLE' DEL # Force delete

?

Show word definition.

'DOUBLE' ? # Shows the definition of DOUBLE

I/O

PRINT Map

Print value to output.

[ 42 ] PRINT # Outputs: 42

Module System

IMPORT

Load a module. Module words use MODULE@WORD path notation.

'music' IMPORT 'json' IMPORT 'io' IMPORT

Available modules:

  • music — SEQ, SIM, PLAY, CHORD, SLOT, GAIN, PAN, ADSR, waveforms (SINE, SQUARE, SAW, TRI)
  • json — PARSE, STRINGIFY, GET, KEYS, SET
  • io — INPUT, OUTPUT
