Control Flow

Conditional (COND)

Evaluates guard/body pairs and executes the first matching branch.

[ 42 ]
  { [ 0 ] < }   { 'negative' }
  { IDLE }      { 'positive' }
  COND

Pipeline Operator (==)

Visual marker for data flow (no-op). Makes complex transformations readable:

[ 1 2 3 4 5 ]
  == { [ 2 ] * } MAP
  == { [ 5 ] < NOT } FILTER
  == [ 0 ] { + } FOLD

Nil Coalescing (=>)

Returns alternative value when first value is NIL:

NIL => [ 0 ]       # Result: [ 0 ]
[ 42 ] => [ 0 ]    # Result: [ 42 ]

# Useful for safe defaults
'key' LOOKUP => [ 'not found' ]

Force Execution (!)

Allows DEL/DEF of dependent words:

# Define dependent words
{ HELPER } 'MAIN' DEF
{ [ 1 ] } 'HELPER' DEF

# Cannot delete HELPER normally
'HELPER' DEL    # Error!

# Force delete
! 'HELPER' DEL  # Success

Best Practices

  • Prefer higher-order functions (MAP, FILTER, FOLD) for data transformations
  • Avoid deep recursion - use FOLD for large data
