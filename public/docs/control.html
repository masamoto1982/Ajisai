Control Flow

ROUTE — Flow Routing

ROUTE routes the flow through condition-action code block pairs. Use modifiers to switch between branching and looping:

cond | action | default | ROUTE        # . ROUTE: single branch (if-else)
cond | action | .. ROUTE               # .. ROUTE: loop (while)
cond | action | default | ,, ROUTE     # ,, ROUTE: bifurcation (keep flow)
cond | action | .. ,, ROUTE            # .. ,, ROUTE: loop + bifurcation

Example: Absolute Value (. ROUTE)

|
  ,, [ 0 ] < |   [ -1 ] * |
                    |
  ROUTE
| 'ABS' DEF

[ -5 ] ABS    # Result: [ 5 ]
[ 3 ] ABS     # Result: [ 3 ]

Example: Loop (.. ROUTE)

# Double until > 1000
[ 1 ]
  ,, [ 1000 ] < |  [ 2 ] * |
.. ROUTE
# Result: [ 1024 ]

# Count up to 100
[ 0 ]
  ,, [ 100 ] < |  [ 1 ] + |
.. ROUTE
# Result: [ 100 ]

Example: Bifurcation (,, ROUTE)

[ 5 ]
  ,, [ 0 ] < NOT | [ 2 ] * |
,, ROUTE
# Result: [ 5 ] [ 10 ]  (original + result)

Pipeline Operator (==)

Visual marker for data flow (no-op). Makes complex transformations readable:

[ 1 2 3 4 5 ]
  == [ 2 ] * | MAP
  == [ 5 ] < NOT | FILTER
  == [ 0 ] + | FOLD

Nil Coalescing (=>)

Returns alternative value when first value is NIL:

NIL => [ 0 ]       # Result: [ 0 ]
[ 42 ] => [ 0 ]    # Result: [ 42 ]

# Useful for safe defaults
'key' LOOKUP => [ 'not found' ]

WAIT - Delayed Execution

'word_name' [ milliseconds ] WAIT

Examples

# Execute after 1 second
[ 'Done!' ] PRINT | 'NOTIFY' DEF
'NOTIFY' [ 1000 ] WAIT

Use cases:

  • Animation and visual effects
  • Debouncing user input
  • Scheduled operations

Force Execution (!)

Allows DEL/DEF of dependent words:

# Define dependent words
HELPER | 'MAIN' DEF
[ 1 ] | 'HELPER' DEF

# Cannot delete HELPER normally
'HELPER' DEL    # Error!

# Force delete
! 'HELPER' DEL  # Success

Recursion

Recursive patterns via word definitions:

Factorial

|
  >> [ 1 ] <=
  >> [ 1 ]
  >>> [ 1 ] - FACTORIAL *
| 'FACTORIAL' DEF

[ 5 ] FACTORIAL    # Result: [ 120 ]

Note: For large iterations, prefer FOLD:

# 5! using FOLD (no recursion)
[ 1 ] [ 5 ] RANGE [ 1 ] '*' FOLD    # Result: [ 120 ]

Best Practices

  • Prefer higher-order functions over explicit loops
  • Keep guard conditions simple and readable
  • Name complex predicates for clarity
  • Avoid deep recursion - use FOLD for large data
