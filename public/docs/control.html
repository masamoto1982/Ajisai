Control Flow

Conditional (COND) (Form)

Evaluates guard/body pairs in order. Executes the body of the first guard that returns true. Use { IDLE } as an else clause that always matches.

[ 42 ]
  { [ 0 ] < }   { 'negative' }
  { [ 0 ] = }   { 'zero' }
  { IDLE }      { 'positive' }
  COND

Rules

  • Guard and body must be paired
  • { IDLE } guard acts as else clause (always matches)
  • No matching guard and no else → CondExhausted error

Pipeline Operator (==)

Visual marker for data flow (no-op). Makes multi-step transformations readable:

[ 1 2 3 4 5 ]
  == { [ 2 ] * } MAP
  == { [ 5 ] < NOT } FILTER
  == [ 0 ] { + } FOLD

Nil Coalescing (=>)

Returns an alternative value when the first value is NIL. If the value is not NIL, the alternative is ignored:

NIL => [ 0 ]       # Result: { 0 }
[ 42 ] => [ 0 ]    # Result: { 42 }

# Combine with FILTER
[ 1 2 3 ] { [ 10 ] < NOT } FILTER => [ 0 ]   # { 0 }

Safe Mode (~)

Errors become NIL instead of halting execution. Useful for graceful error handling:

[ 1 2 3 ] [ 10 ] ~ GET            # NIL (instead of index error)
[ 1 2 3 ] [ 10 ] ~ GET => [ 0 ]   # { 0 } (safe + default)

# "No Change is Error" also caught
[ 1 ] ~ REVERSE                    # NIL (no change → NIL)

Combine modifiers freely (order-independent):

1 2 3 [ 10 ] ~ ,, .. GET          # 1 2 3 NIL

Force Execution (!)

Allows DEL/DEF of dependent words:

{ [ 2 ] * } 'DOUBLE' DEF
{ DOUBLE DOUBLE } 'QUAD' DEF

# Cannot delete DOUBLE normally (QUAD depends on it)
'DOUBLE' DEL                   # Error!

# Force delete
! 'DOUBLE' DEL                # Success (with warning)

Code Blocks

{ code } or ( code ) define deferred code. Used for DEF, MAP, FILTER, FOLD, COND.

{ [ 2 ] * } 'DOUBLE' DEF
[ 1 2 3 ] { [ 10 ] + } MAP

Code blocks are first-class values (CodeBlock variant in ValueData).

Metaprogramming

  • EXEC — Execute a vector as code: [ [ 2 ] [ 3 ] * ] EXEC → { 6 }
  • EVAL — Parse and execute a string: '[ 2 ] [ 3 ] *' EVAL → { 6 }

Best Practices

  • Prefer higher-order functions (MAP, FILTER, FOLD) over deep word chains
  • Call depth is limited to 5 levels (main + 4 chain steps)
  • Use ~ safe mode + => nil coalescing for robust pipelines
  • Use == pipeline operator for readable multi-step transformations
