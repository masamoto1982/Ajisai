Ajisai チュートリアル

1. 新しい設計思想：スタックはVectorである

Ajisaiの最も重要なコンセプトは、「**スタックの実体はVectorである**」というものです。これにより、Vectorを操作するワードは、そのままスタック全体を操作するためにも使えます。この統一された考え方が、Ajisaiのシンプルさとパワフルさの源泉です。

この考え方を実現するため、多くの組み込みワードは操作対象を切り替えることができます。

操作対象の指定子: STACKTOP と STACK

ワードがどこに対して作用するかを明示的に指定するための「指定子ワード」が導入されました。

  • STACKTOP: ワードの操作対象を、スタックの一番上にあるVectorの**内部要素**に設定します。これは最も一般的な操作です。
  • STACK: ワードの操作対象を、**スタックそのもの**（スタック上に積まれている全Vector）に設定します。
デフォルト動作: 指定子ワードを省略した場合、Ajisaiは自動的に STACKTOP が指定されたものとして動作します。これにより、従来のコードの簡潔さは維持されます。
# これはSTACKTOPを指定した場合と同じ (デフォルト動作) [ 10 20 30 ] [ 1 ] GET # -> Workspace: [ 20 ] # 明示的にSTACKTOPを指定 [ 10 20 30 ] STACKTOP [ 1 ] GET # -> Workspace: [ 20 ] # STACKを指定して、スタックそのものを操作する [ 10 ] [ 20 ] [ 30 ] STACK [ 1 ] GET # -> Workspace: [ 10 ] [ 20 ] [ 30 ] [ 20 ] (スタックの1番目(20)をコピー)

2. 基本的なデータ型

Ajisaiが扱うデータは、すべてVectorという形式で表現されます。

  • 数値: [ 10 ], [ 3/4 ], [ 1.5e3 ] のように記述します。内部的にはすべて分数として扱われるため、小数や巨大数も誤差なく計算できます。
  • 文字列: [ 'Hello' ] のようにシングルクォートで囲みます。
  • 真偽値: [ TRUE ] または [ FALSE ] です。
  • Nil: [ NIL ] は何もないことを示す値です。
  • Vector: [ 1 2 'Apple' ] のように、複数の値をまとめて格納できます。Vectorの中にVectorを入れる（ネストする）ことも可能です。

3. Vectorとスタックの操作

以下のワードは、STACKTOP (デフォルト) と STACK の両方で動作します。

GET: 要素の取得

# STACKTOP (デフォルト): Vectorの要素を取得 [ 1 2 3 ] [ 0 ] GET # -> [ 1 ] # STACK: スタック上のVectorを取得 (伝統的なPICK) [ 1 ] [ 2 ] [ 3 ] STACK [ 0 ] GET # -> [ 1 ] [ 2 ] [ 3 ] [ 3 ]

REMOVE: 要素の削除

# STACKTOP (デフォルト): Vectorの要素を削除 [ 1 2 3 ] [ 1 ] REMOVE # -> [ 1 3 ] # STACK: スタック上のVectorを削除 (伝統的なDROP) [ 1 ] [ 2 ] [ 3 ] STACK [ 0 ] REMOVE # -> [ 1 ] [ 2 ]

REVERSE: 順序の反転

# STACKTOP (デフォルト): Vectorの要素を反転 [ 1 2 3 ] REVERSE # -> [ 3 2 1 ] # STACK: スタック上の指定個数のVectorを反転 (伝統的なSWAP/ROT) [ 1 ] [ 2 ] [ 3 ] STACK [ 2 ] REVERSE # -> [ 1 ] [ 3 ] [ 2 ] (SWAP)
位置の指定は0オリジン、量の指定は1オリジン
インデックス番号で「何番目」を指定する場合は 0 から始まります。一方、「何個」のように量を指定する場合は 1 から数えます。

4. 高階関数

高階関数もまた、STACKTOP と STACK の両方で動作します。

MAP: 各要素への適用

[ 2 ] * 'DOUBLE' DEF # STACKTOP (デフォルト): Vectorの各要素に 'DOUBLE' を適用 [ 1 2 3 ] 'DOUBLE' MAP # -> [ 2 4 6 ] # STACK: スタック上の3つのVectorそれぞれに 'DOUBLE' を適用 [ 1 ] [ 2 ] [ 3 ] STACK [ 3 ] 'DOUBLE' MAP # -> [ 2 ] [ 4 ] [ 6 ]

5. カスタムワード（関数の定義）

DEF を使うと、一連の処理をまとめて新しいワードとして定義できます。この機能は設計変更の影響を受けません。

# 与えられた数値を二乗するワード 'SQUARE' を定義 [ DUP * ] 'SQUARE' '数を二乗する' DEF # 定義したワードを使ってみる [ 5 ] SQUARE
